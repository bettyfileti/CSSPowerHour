CSS Power Hour

Before CSS Styling

Design or Development

When diagnosing Web Design problems, you want to first determine the what and where of the problem. You might even have multiple problems going on at the same time!

Here's What You Do

  • Ask yourself if you can clearly say what you are trying to do, in a clear way.
  • Grab a piece of paper, or open Excalidraw or Figma. Then, sketch.
  • Research similar web design patterns. Read Nielsen Norman Group

It's also ok to use CSS and HTML to test and sketch. Just remember that you might accidentally insert problems into your code structure.

CSS Problems

Once you know what you are trying to do, you want to determine where the problem is taking place. It could be the CSS, the HTML structure, or your project folder structure.

Here's What You Do

  • Go to your CSS file, and add a red background to the body element.
    body {
     background-color: red;
    }
  • Add a red border to the element you are trying to target.
    border: 1px solid red
  • Compare what you expect with what is happening.

Keep things simple and easy to identify. Make heavy use of the browser developer tools.

Layout, Spacing, & Targeting

When debugging CSS, it can be helpful to articulate what the problem looks like, as a way of discovering your solution. This will also help you when googling or asking for help.

Here's What You Do

  • Is it a layout problem?
    "I want this to be over here, but it keeps going there."
  • Is it a spacing problem?
    "I can't figure out why this is so far away from this."
  • Is it a targeting problem?
    "When I change this, these things also break!"

Try to tackle these issues one by one. Change one thing at a time in a predictable manner, get it working, then move on to the next task.
